# 规范：Agent Plus 自我监督与校验机制

## Why

当前Agent Plus存在以下问题：
1. **缺少自我校验**：Agent完成任务后直接标记完成，不验证成果是否符合需求
2. **监督机制未集成**：Reviewer和Supervisor组件存在但未在主循环中使用
3. **无自动修复能力**：发现问题时无法自动调整重试

## 目标

为Agent Plus补充：
1. 任务完成前的成果校验机制
2. 集成Reviewer进行实时审查
3. 发现问题时的自动修复循环
4. 质量门禁：只有通过校验才能标记完成

## 技术方案

### 1. 成果校验机制

在`complete_task`工具执行前，自动触发成果校验：
- 检查生成的文件是否存在
- 检查文件内容是否符合基本要求
- 检查功能是否完整实现

### 2. Reviewer集成

在Agent主循环中集成Reviewer：
- 每次工具调用前审查
- 每次工具调用后检查结果
- 发现问题时注入修正指令

### 3. 自动修复循环

当校验失败时：
1. 记录失败原因
2. 生成修复指令
3. 注入到消息历史
4. 继续执行直到成功或达到重试上限

### 4. 质量门禁

修改`complete_task`工具：
- 执行前必须通过校验
- 校验失败则返回错误信息
- Agent需要修复问题后重新尝试

## 实现范围

### 文件修改

1. `internal/agent/core.go` - 集成Reviewer和校验逻辑
2. `internal/tools/state_tools.go` - 增强complete_task工具
3. `internal/agent/verifier.go` - 新增成果校验器
4. `internal/agent/prompts.go` - 添加校验相关提示词

### 新增组件

1. **Verifier** - 成果校验器
   - 文件存在性检查
   - 内容完整性检查
   - 需求匹配度检查

2. **SelfCorrector** - 自我修正器
   - 分析校验失败原因
   - 生成修复指令
   - 管理重试次数

## 验收标准

1. Agent完成任务前自动校验成果
2. 校验失败时自动尝试修复
3. 达到重试上限时报告失败原因
4. 所有测试通过

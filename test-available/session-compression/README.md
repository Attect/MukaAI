# 会话压缩功能可行性测试

> 测试时间：2026-03-12  
> 状态：待执行

## 测试目的

验证会话压缩功能的两种触发方式和压缩策略的可行性：

1. **自动压缩（AUTO）**: 系统根据 token 阈值自动触发，压缩后自动发送"继续"指令
2. **用户主动压缩（USER_INITIATED）**: 用户主动触发，压缩后等待用户继续
3. **用户消息触发压缩**: 用户消息导致 token 超限时的特殊处理流程

## 测试环境

- **AI 服务**: OpenAI 格式 API
- **地址**: http://127.0.0.1:11451
- **模型**: jaahas/qwen3.5-uncensored:9b
- **认证**: 无需 key

## 测试内容

### 1. 压缩提示词测试

测试两种不同的压缩提示词模板是否能够产生符合预期的摘要：

- **自动压缩提示词**: AI 口吻，结尾体现"继续"语义
- **用户主动压缩提示词**: 客观描述，结尾体现"等待用户回应"语义

### 2. 压缩后处理逻辑测试

- 自动压缩后是否能够正确触发"继续"指令
- 用户主动压缩后是否正确等待用户输入

### 3. 用户消息触发压缩测试

- 模拟用户消息导致 token 超限的场景
- 验证先压缩再处理消息的流程

## 测试步骤

1. 启动测试脚本
2. 观察压缩提示词的效果
3. 验证压缩后的行为是否符合预期
4. 记录测试结果

## 预期结果

1. ✅ 自动压缩提示词生成 AI 口吻的摘要，结尾体现连续性
2. ✅ 用户主动压缩提示词生成客观描述的摘要，结尾体现等待回应
3. ✅ 压缩后处理逻辑正确（自动压缩自动继续，用户压缩等待用户）
4. ✅ 用户消息触发压缩时，先压缩再处理消息

# JavaScript 编程测试题目

## 概述

本测试包含三个复杂的JavaScript编程题目，主要考察异步处理与数据处理能力。所有题目均为纯Node.js程序，不涉及UI依赖。

## 技术要求

- **Node.js版本**: 18+
- **模块系统**: ES模块语法 (ESM)
- **第三方库**: 不允许使用
- **代码风格**: 清晰、可维护、符合最佳实践

## 题目列表

### [Task 1: 异步任务调度器](./task1-task-scheduler/README.md)
实现一个功能完整的异步任务调度器，支持定时任务、延迟任务、周期任务，具备优先级管理和错误恢复机制。

**难度**: ⭐⭐⭐⭐
**知识点**: Promise、异步编程、定时器、事件循环

### [Task 2: 流式数据处理器](./task2-stream-processor/README.md)
实现一个高性能的流式数据处理器，支持多种数据转换操作，具备背压控制和错误恢复能力。

**难度**: ⭐⭐⭐⭐⭐
**知识点**: Stream API、背压控制、数据处理、内存管理

### [Task 3: 有限状态机框架](./task3-state-machine/README.md)
实现一个功能强大的有限状态机框架，支持状态定义、转换规则、动作触发，具备状态历史和并行状态处理能力。

**难度**: ⭐⭐⭐⭐
**知识点**: 状态机设计、事件驱动、状态管理

## 验收标准

每个题目都包含详细的验收标准，请确保：

1. **功能完整性**: 实现所有要求的功能
2. **代码质量**: 代码清晰、注释完善、符合规范
3. **测试覆盖**: 提供充分的测试用例
4. **性能要求**: 满足性能指标要求
5. **错误处理**: 正确处理各种异常情况

## 提交要求

1. 每个任务在对应目录下实现
2. 提供完整的实现代码和测试代码
3. 编写README说明实现思路和关键点
4. 确保所有测试用例通过

## 评分维度

- **功能实现** (40%): 是否完整实现所有要求的功能

- **测试覆盖** (20%): 测试用例的完整性和有效性
